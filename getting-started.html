Getting Started

This guide will help you get started with the Responsive Starter Kit Pro. It covers how to download the kit, the overall directory/file structure, and a brief overview of the layout of a basic page. It also includes code samples to help you include some helpful add-ons.

Download

The source code for the starter kit is available in a GitHub repository. You can begin by heading over there and cloning or forking the repository directly. Or if you'd prefer, you can download a ZIP archive of the starter kit if you'd prefer to get a snapshot of the code without worrying about future updates. Use one of the following links to get started.

What's Included

The starter kit was built with a basic toolkit requiring minimal setup. The HTML files can be viewed directly in your browser, but if you'd like to test in IE 8, you'll need to view them from a local web server such as MAMP, WAMP, simple Apache, or similar.

The CSS was written with the LESS CSS pre-processor, so you'll also need a compiler such as Less.app or CodeKit for Mac OS X or SimpLESS or WinLess for Windows. Although you can hack away at the compiled CSS directly, you'll really be missing out on the benefits of a CSS pre-processor, so it's not recommended to head down that path. In fact, you should always edit the LESS files instead of the CSS files. Editing the CSS files directly and then editing the LESS file later will cause you to lose any changes you'd previously made directly to the CSS file. There are a few CSS files that are included with jQuery plugins. Those are fair game to be edited directly.

Note: If you're not already familiar with CSS pre-processors, head over to the LESS website to see what all the fuss is about. Trust me, it's not a waste of time to use a CSS pre-processor. And there's no time like the present to get started if you haven't already.

Once you've grabbed the source code and uncompressed it to your hard drive, it should look like this.

/
└── assets/
    ├── css/
    ├── img/
    │   ├── fancybox
    │   ├── flexslider
    │   └── ico
    └── js/

There are a handful of files in the base directory of the project.

  • README.md – This is a GitHub document with general information about the starter kit.
  • index.html – This is the introductory page for the project. It includes links to sample page layouts, etc.
  • layout*.html – There's a variety of sample layout pages which will demonstrate typical page layouts, including sidebars on the left/right, two sidebars, full width, and more. These can be used as a good starting point for your project. Adjust as needed.
  • style-guide.html – The style guide page contains a sampling of headings, paragraphs, unordered and ordered lists, inline images with content, columns (grids), alerts, and buttons.
  • add-ons.html – The Add-ons page contains an assortment of useful responsive components that can be added for additional functionality, such as content carousels, tabbed navigation, responsive data tables, etc.

All of the style sheets are included in the assets/css directory.

  • /assets/css – The css directory contains all of the .less files as well as the compiled .css files.

Images are included in the assets/img directory.

  • /assets/img – Image files can be placed here. Create additional sub-directories as needed. There are a few placeholder images which can be deleted.
  • /assets/img/fancybox – Images used by the Fancybox modal plugin.
  • /assets/img/flexslider – Images used by the Flexslider content carousel (slider) plugin.
  • /assets/img/ico – General purpose placeholder icons.

And finally, JavaScript files are contained in the assets/js directory.

  • /assets/js – JavaScript files, including sample global and section/page files are included here. You'll also find a small handful of jQuery plugin files.

Basic Structure of a Page

There's an empty layout page (layout-empty.html) which contains the most basic elements to start a page. Duplicate this file and open it in your text editor. Here's a simplified version of that page to illustrate its basic structure.

<!doctype html>
<html>
<head>
    <meta charset="UTF-8">
    <title>Page Title</title>    
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <link rel="stylesheet" href="assets/css/main.css">
    <link rel="stylesheet" href="assets/css/layout-sidebar-left.css">
    <link rel="stylesheet" href="assets/css/theme.css">
    <!-- Additional items go here, such as meta tags, scripts, web fonts, and icons. -->
</head>
<body>

<div class="site-container">

    <header role="banner" class="site-header">
        <!-- Header content goes here, such as site branding and primary nav. -->
    </header><!-- /.site-header -->

    <div class="site-content">
        <div class="page-content">
            <!-- Page-specific content goes here, such as a breadcrumb trail. -->

            <main role="main">
                 <h1>Page Title</h1>
                <!-- Main content goes here. -->
            </main>

            <aside role="complementary" class="sidebar">
                <!-- Sidebar content goes here. -->
            </aside><!-- /.sidebar -->

        </div><!-- /.page-content -->
    </div><!-- /.site-content -->

    <footer role="contentinfo" class="site-footer">
        <!-- Footer content goes here. -->
    </footer><!-- /.site-footer -->

</div><!-- /.site-container -->

</body>
</html>

There are additional items in the base template, but this should give you and idea of the overall structure of a typical page. Move things around if you'd like. Take a look at all the sample layout pages and see which one is the best starting point for your project. Make adjustments, additions, and subtractions to the overall structure. Make it your own.

Remember: All of this structure is editable to match your project's design. This is just a starting point.

Code Samples

The Add-ons page contains a handful of useful items that can be added to a page to provide some additional interactivity, such as content carousels, tabbed navigation, and much more. The following code samples (snippets) will help you get started building these add-ons into your pages.

Modal Dialogs

Modal dialogs are created with the jQuery plugin fancyBox. Additional details are available on their site.

Place in the head of your document:

<link rel="stylesheet" href="assets/css/jquery.fancybox.css">

Place in the foot of your document:

<script src="assets/js/jquery.fancybox.pack.js">
<script src="assets/js/jquery.fancybox-media.js"></script>

Place in the global.js your document:

$(document).ready(function () {
    // modal for general content (fancyBox)
    if ($(".fancybox").length > 0) {
        $(".fancybox").fancybox({
            maxWidth: 800,
            maxHeight: 600
        });
    }

    // modal for video player (fancyBox)
    if ($('.fancybox-media').length > 0) {
        $('.fancybox-media').fancybox({
            width: 800,
            height: 450,
            padding: 2,
            openEffect: 'none',
            closeEffect: 'none',
            helpers: {
                media: {}
            }
        });
    }
});

Use this HTML code to add an inline modal dialog:

<a href="#modal-content" class="fancybox">modal inline</a>

<div id="modal-content" style="display: none;">
    Content goes here.
</div>

Use this HTML code to add a modal image:

<a href="image.jpg" title="Caption Goes Here" class="fancybox">modal image</a>

Use this HTML code to add a modal gallery of images. Notice how the rel attributes are all the same. This is what ties the images together in the modal dialog so that the user can cycle through the full set.

<div class="image-gallery outline-pad">
    <a class="fancybox" rel="gallery1" href="large-image-1.jpg" title="Caption Text Goes Here"><img src="thumbnail-image-1.jpg" alt=""></a>
    <a class="fancybox" rel="gallery1" href="large-image-2.jpg" title="Caption Text Goes Here"><img src="thumbnail-image-2.jpg" alt=""></a>
    <a class="fancybox" rel="gallery1" href="large-image-3.jpg" title="Caption Text Goes Here"><img src="thumbnail-image-3.jpg" alt=""></a>
</div>

Use this HTML code to add a modal video. Replace the href attribute with your video's URL.

<a href="http://player.vimeo.com/video/50905661" class="fancybox fancybox-media">modal video</a>
